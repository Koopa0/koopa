# AI Pipeline 情境規劃

> 這份文件列出所有 AI Pipeline 可實現的情境，作為後續 Genkit Flow 設計和前端功能規劃的基礎。

---

## 系統定位

koopa0.dev 不只是一個部落格，而是一個**個人知識飛輪**：

```
學習 (Obsidian)
  ↓
整理 (AI 審核 + 潤色)
  ↓
發佈 (Blog)
  ↓
追蹤 (產出頻率 + 內容分佈)
  ↓
分析 (知識缺口 + 內容策略)
  ↓
規劃 (建議下一步學什麼/寫什麼)
  ↓
回到學習
```

AI 的角色是**處理層**——把人類不想手動做的重複性工作自動化，把人類看不到的模式浮現出來。

## 情境總覽

### A. 內容品質（Obsidian → Blog）

| # | 情境 | 描述 | 觸發方式 |
|---|------|------|---------|
| A1 | 文章審核 | AI 檢查錯字、語法、技術錯誤，決定能不能直接發佈還是進 review queue | Webhook sync 後自動 |
| A2 | Excerpt 生成 | 自動產出 2-3 句摘要，存入 content.excerpt | Webhook sync 後自動 |
| A3 | Reading Time 估算 | 根據字數和內容複雜度估算閱讀時間 | Webhook sync 後自動 |
| A4 | Tag/Topic 建議 | 根據內容分析，補足缺漏的分類標籤 | Webhook sync 後自動 |
| A5 | 筆記 → 文章潤色 | 把 Obsidian 概念筆記（callout、知識連結風格）改寫成適合對外發佈的文章語氣 | 手動觸發 / review 時選擇 |
| A6 | SEO 標題/描述建議 | 幫文章生成更好的標題或 meta description | 發佈前建議 |

### B. 自動內容生成

| # | 情境 | 描述 | 觸發方式 |
|---|------|------|---------|
| B1 | Build Log 生成 | 從 Projects 的完成 tasks 彙整成開發紀錄 | 週期性 / 手動觸發 |
| B2 | 週報/月報 Digest | 彙整一段時間的 contents 生成摘要 | 週期性（每週日 / 每月底） |
| B3 | Bookmark 生成 | 讀完外部文章後，AI 幫你生成摘要 + 你加評語 → 變成 bookmark content | Reading List 完成時 |
| B4 | TIL 生成 | 從當天的 Obsidian 筆記或 Notion tasks 提取今日學到的東西 | 每日自動 / 手動觸發 |

### C. Notion Ultimate Brain 整合

| # | 情境 | 描述 | 觸發方式 |
|---|------|------|---------|
| C1 | Projects 雙向同步 | Notion 專案狀態 ↔ Blog projects 表保持一致 | Notion webhook / 定時 poll |
| C2 | Tasks 完成追蹤 | 記錄每天/每週完成了哪些任務，存入 tracking 系統 | Notion webhook |
| C3 | AI 主動安排任務 | 根據目標、deadline、歷史完成模式，建議今天該做什麼 | 每日早上 / 手動觸發 |
| C4 | 執行率分析 | 追蹤計劃 vs 實際完成，分析瓶頸和模式 | 週期性分析 |
| C5 | Reading List 追蹤 | 讀完標記 → 觸發 B3 bookmark 生成 | Notion webhook |
| C6 | 週期回顧提醒 | 每週五提醒做 weekly review，附上本週數據摘要 | 定時觸發 |

### D. 外部資料收集

| # | 情境 | 描述 | 觸發方式 |
|---|------|------|---------|
| D1 | RSS 訂閱收集 | 自動抓關注的技術 blog 新文章 | 定時 poll |
| D2 | AI 篩選評分 | 收集進來的資料 AI 判斷值不值得看，評分排序 | 收集後自動 |
| D3 | 自動摘要 | 外部文章自動生成中文摘要 | 收集後自動 |

### E. 個人效率系統

| # | 情境 | 描述 | 觸發方式 |
|---|------|------|---------|
| E1 | 每日任務建議 | AI 根據待辦、優先級、歷史習慣推薦今天的任務清單 | 每日早上 |
| E2 | 目標追蹤 | 設定季度/月度目標，AI 追蹤進度並提醒偏離 | 週期性 |
| E3 | 知識缺口分析 | 根據內容和學習記錄，建議下一步該學什麼 | 週期性 / 手動觸發 |
| E4 | 時間分配分析 | 分析花在哪些 topic 的時間，跟目標是否一致 | 週期性 |

### F. 學習與成長

| # | 情境 | 描述 | 觸發方式 |
|---|------|------|---------|
| F1 | 知識圖譜視覺化 | 分析 Obsidian 筆記的 `[[連結]]` 關係，看見知識結構中的群聚和孤立區 | 手動觸發 / 同步後更新 |
| F2 | 刻意練習建議 | 發現你寫了很多理論筆記但沒有實戰 → 建議做一個 benchmark/實作 | 週期性分析 |
| F3 | 費曼檢測 | AI 讀你的筆記，問你問題來測試是否真正理解，而不只是整理過 | 手動觸發 |
| F4 | 跨領域連結發現 | 發現不同 topic 的筆記有共通概念 → 建議寫比較文或統整文 | 週期性分析 |

### G. 對外影響力

| # | 情境 | 描述 | 觸發方式 |
|---|------|------|---------|
| G1 | 內容策略建議 | 根據現有文章分佈，建議「你的 X 系列缺一篇 Y」 | 週期性 / 手動觸發 |
| G2 | 系列文規劃 | 分析某個 topic 的筆記群，建議組成系列文的大綱 | 手動觸發 |
| G3 | 發佈節奏追蹤 | 追蹤多久沒發文，提醒 pipeline 裡有幾篇 draft 可以完成 | 週期性 |
| G4 | 讀者視角審核 | AI 假裝目標讀者（如 Go 中級開發者），回饋「這段看不懂」「需要範例」 | 手動觸發 / 發佈前 |

### H. 專案管理

| # | 情境 | 描述 | 觸發方式 |
|---|------|------|---------|
| H1 | Commit → 進度更新 | Push code 到 project repo → AI 分析 commit messages → 自動更新 project 進度描述 | GitHub webhook |
| H2 | README 生成 | 根據 project 的 commits、技術棧、筆記，維護 project 的展示頁描述 | 手動觸發 |
| H3 | 技術決策記錄 | 開發中做了選擇（用 pgx 不用 gorm），AI 幫你寫成 ADR 格式存起來 | 手動觸發 |

### I. 日常工作流

| # | 情境 | 描述 | 觸發方式 |
|---|------|------|---------|
| I1 | Morning Brief | 每天早上推一份摘要：今天任務、到期 deadline、昨天未完成、draft 狀態 | 每日早上定時 |
| I2 | Context Switch 輔助 | 中斷任務後回來時，AI 給你「上次做到這裡，接下來要做 X」 | 手動觸發 |
| I3 | 週五自動回顧 | 自動生成本週做了什麼、學了什麼、寫了什麼、哪些目標有進展 | 每週五定時 |
| I4 | 碎片時間利用建議 | 有 30 分鐘空檔 → 建議「把這篇 draft excerpt 寫完」或「review 這篇收集的文章」 | 手動觸發 |

## 技術實現基礎

### Genkit Flow 架構

每個情境對應一個或多個 Genkit Flow：

```
觸發源                    Genkit Flow              輸出
──────                    ──────────              ────
GitHub Webhook     ──→   content-review-flow   ──→  review queue / 直接發佈
定時 Cron          ──→   digest-gen-flow       ──→  draft content
Notion Webhook     ──→   task-sync-flow        ──→  tracking 記錄
手動 API 呼叫      ──→   series-plan-flow      ──→  建議回應
```

### 共用能力

多個情境會共用的底層能力：

| 能力 | 使用者 |
|------|--------|
| 文章摘要生成 | A2, B2, B3, D3 |
| 內容分類/標籤 | A4, D2 |
| 中文文字校對 | A1, A5 |
| Notion API 讀寫 | C1-C6, E1-E4 |
| 進度追蹤/統計 | C4, E2, E4, G3 |
| Prompt 模板管理 | 全部 |

### 資料流向

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│   輸入源     │     │  Genkit Flow │     │   輸出目標   │
│             │     │              │     │             │
│ Obsidian ───┼──→──┤ 審核/潤色    ├──→──┤ contents 表  │
│ Notion ─────┼──→──┤ 同步/分析    ├──→──┤ projects 表  │
│ RSS ────────┼──→──┤ 收集/篩選    ├──→──┤ collected 表 │
│ GitHub ─────┼──→──┤ 追蹤/生成    ├──→──┤ tracking 表  │
│ 手動觸發 ───┼──→──┤ 規劃/建議    ├──→──┤ review 表    │
│ 定時 Cron ──┼──→──┤ 週期性任務   ├──→──┤ Notion API   │
└─────────────┘     └──────────────┘     └─────────────┘
```
